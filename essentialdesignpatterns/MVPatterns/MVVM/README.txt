Introduction:

this document will exemplify the use MVVM pattern a pattern that provide an architecture pattern based on teh MVC and MVP. which attemps to more clearly separate the development of user-interface (UI) from that of the business logic and behavior in an application.

To this end, many implementations of this pattern make use of declarative data bindings to allow a separation of work on Views from other layers.


History:

now recently the MVVM pattern has been implemented in Javascript in the form of structural frameworks such as KnockoutJS, Kendo MVVM, and Knockback.js.
